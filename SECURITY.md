# Security Policy

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Please report security issues by email to **security@skybyte.dev**. Include as much detail as possible: steps to reproduce, affected versions, and any proof-of-concept.

## Response Timeline

| Milestone | Target |
|---|---|
| Acknowledgement | Within 72 hours |
| Status update | Within 7 days |
| Patch release | Within 14 days (critical), 30 days (other) |

We will coordinate disclosure timing with you and credit you in the release notes if you wish.

## Scope

This policy covers the WPPlugin Watch WordPress plugin (`wppluginwatch`). The backend API at `api.wpplugin.watch` is operated by Skybyte Hosting LLC and is also in scope — please use the same email address to report issues there.
